Task 2

Configure HQ MATARAM Router

  1. Configure a hostname router with name “HQ”.
Router>en
Router#conf t
Enter configuration commands, one per line. End with CNTL/Z.
Router(config)#hostname HQ
  1. Configure the encapsulation protocol on Serial0/0/0 interface that connected to the ISP router with PPP authentication using CHAP with the password “LksNTB”.
HQ(config)#int s0/0/0
HQ(config-if)#encapsulation ppp
HQ(config-if)#ppp authentication chap 
HQ(config)#username ISP password LksNTB
  1. Configure the IP address on the Serial0/0/0 interface that connected to the ISP router using a 2nd IP address from the HQ-ISP subnet address 192.0.2.16/29.
HQ(config)#int s0/0/0
HQ(config-if)#ip address 192.0.2.18 255.255.255.248
HQ(config-if)#no shut
%LINK-5-CHANGED: Interface Serial0/0/0, changed state to up
%LINEPROTO-5-UPDOWN: Line protocol on Interface Serial0/0/0, changed state to up
HQ(config-if)#description TO ISP
  1. Configure the IP address on GigabitEthernet0/1 interface that connected to the HQ Server using a 1st IP address from the HQ-ISP subnet address 10.0.14/30.
HQ(config)#int g0/1
HQ(config-if)#ip address 10.0.14.1 255.255.255.252
HQ(config-if)#no shut
%LINK-5-CHANGED: Interface GigabitEthernet0/1, changed state to up
%LINEPROTO-5-UPDOWN: Line protocol on Interface GigabitEthernet0/1, changed state to up
HQ(config-if)#description TO HQ Server

  1. Configure router-on-stick for communication between VLANs by creating a subinterface on GigabitEthernet0/0 interface that implements IEEE 802.1Q.
    Assign a IP address on each subinterface using the following requirements:

    • Subinterface GigabitEthernet0/0.11 for VLAN 11 HRD with the 1st IP address of subnet 10.0.11.0/24.

    • Subinterface GigabitEthernet0/0.12 for VLAN 12 MARKETING with the 1st IP address of subnet 10.0.12.0/24.

    • Subinterface GigabitEthernet0/0.13 for VLAN 13 NATIVE is set as a native VLAN and uses the 1st IP address from subnet 10.0.13.0/24. 

HQ(config)#int g0/0
HQ(config-if)#description ROUTER-ON-STICK
HQ(config-if)#no shut
%LINK-5-CHANGED: Interface GigabitEthernet0/0, changed state to up
%LINEPROTO-5-UPDOWN: Line protocol on Interface GigabitEthernet0/0, changed state to up

HQ(config-if)#int g0/0.11
%LINK-5-CHANGED: Interface GigabitEthernet0/0.11, changed state to up
%LINEPROTO-5-UPDOWN: Line protocol on Interface GigabitEthernet0/0.11, changed state to up
HQ(config-subif)#description VLAN 11 HRD
HQ(config-subif)#encapsulation dot1Q 11 
HQ(config-subif)#ip address 10.0.11.1 255.255.255.0

HQ(config)#int g0/0.12
%LINK-5-CHANGED: Interface GigabitEthernet0/0.12, changed state to up
%LINEPROTO-5-UPDOWN: Line protocol on Interface GigabitEthernet0/0.12, changed state to up
HQ(config-subif)#description VLAN 12 MARKETING
HQ(config-subif)#encapsulation dot1Q 12
HQ(config-subif)#ip address 10.0.12.1 255.255.255.0

HQ(config)#int g0/0.13
%LINK-5-CHANGED: Interface GigabitEthernet0/0.13, changed state to up
%LINEPROTO-5-UPDOWN: Line protocol on Interface GigabitEthernet0/0.13, changed state to up
HQ(config-subif)#description VLAN 13 NATIVE
HQ(config-subif)#encapsulation dot1Q 13 native 
HQ(config-subif)#ip address 10.0.13.1 255.255.255.0
  1. Create a DHCP Server.

    • Pool name “HRD” for VLAN 11 with a subnet 10.0.11.0/24.

    • Pool name “MARKETING” for VLAN 12 with a subnet 10.0.12.0/24.

    • Pool name “NATIVE” for VLAN 13 with a subnet 10.0.13.0/24. The TCP/IP parameters set on each pool are:

    • Default gateway obtained by DHCP Client uses 1st IP address of each subnet of each VLAN.

    • DNS Server which is 192.0.2.1.

    • Especially for the “NATIVE” pool, the parameter distribution is also carried out by the IP address of the Wireless Controller WLC_HQ, which uses the the 4th IP address subnet 10.0.13.0/24 so the LeighWeight Access Point (LWAP) can register automatically. 

HQ(config)#ip dhcp pool HRD
HQ(dhcp-config)#network 10.0.11.0 255.255.255.0
HQ(dhcp-config)#default-router 10.0.11.1
HQ(dhcp-config)#dns-server 192.0.2.1

HQ(config)#ip dhcp pool MARKETING
HQ(dhcp-config)#network 10.0.12.0 255.255.255.0
HQ(dhcp-config)#default-router 10.0.12.1
HQ(dhcp-config)#dns-server 192.0.2.1

HQ(config)#ip dhcp pool NATIVE
HQ(dhcp-config)#network 10.0.13.0 255.255.255.0
HQ(dhcp-config)#default-router 10.0.13.1
HQ(dhcp-config)#dns-server 192.0.2.1

  • Set the IP address not leased to the DHCP Client for each pool.

    • The 1st IP address VLAN 11 HRD and VLAN 12 MARKETING.
    • The 1st through 4th IP address VLAN 13 NATIVE. 
HQ(config)#ip dhcp excluded-address 10.0.11.1
HQ(config)#ip dhcp excluded-address 10.0.12.1
HQ(config)#ip dhcp excluded-address 10.0.13.1 10.0.13.4
  1. Configure the default route to the ISP using the gateway of the 1st IP address of the HQ-ISP subnet 192.0.2.16/29 used by interface Serial0/0/0. 
HQ(config)#ip route 0.0.0.0 0.0.0.0 192.0.2.17

  1. Create an Extended Named ACL with the name “ALLOW-INTERNET” and has the following requirements:

    • Allow queries to a DNS server with Public IP address 192.0.2.1 from any source to map domain names to IP address and vice versa

    • Allow Internet access ONLY to the email service provided by gmail.com with a Public IP address of 192.0.2.5 for all hosts on VLAN 11 HRD which has a subnet address of 10.0.11.0/24.
      Make sure that the hosts on that VLAN can only send and download emails from gmail.com.

    • Allow Internet access for all hosts on VLAN 12 MARKETING and VLAN 13 NATIVE with subnet 10.0.12.0/24 and 10.0.13.0/24.

HQ(config)#ip access-list extended ALLOW-INTERNET
HQ(config-ext-nacl)#permit udp any host 192.0.2.1 eq 53 (Query DNS) 
HQ(config-ext-nacl)#permit tcp 10.0.11.0 0.0.0.255 192.0.2.5 0.0.0.0 eq 25  (Email)
HQ(config-ext-nacl)#permit tcp 10.0.11.0 0.0.0.255 192.0.2.5 0.0.0.0 eq 110 (Email) 
HQ(config-ext-nacl)#permit tcp 10.0.12.0 0.0.0.255 0.0.0.0 255.255.255.255  (Internet)
HQ(config-ext-nacl)#permit tcp 10.0.13.0 0.0.0.255 0.0.0.0 255.255.255.255  (Internet)
  1. Configure NAT Overload or Port Address Translation (PAT) for the Standard Named ACL “ALLOW-INTERNET”.
HQ(config)#int s0/0/0
HQ(config-if)#ip nat outside
HQ(config)#int ra g0/0.11 - g0/0.13
HQ(config-if-range)#ip nat inside
HQ(config)#ip nat inside source list ALLOW-INTERNET interface s0/0/0 overload
  1. Configure static NAT so that ONLY HTTP and HTTPS services on the HQ server with Private IP address 10.0.14.2 are translated to Public IP address 192.0.2.19.
HQ(config)#ip nat inside source static 10.0.14.2 192.0.2.19
HQ(config)#int g0/1
HQ(config-if-range)#ip nat inside
  1. Configure the HQ router to synchronize the time with an NTP Server that has an IP address of 192.0.2.2 and uses md5 message authentication with key 46 and password “NTBHebat2021”.
HQ(config)#ntp server 192.0.2.2 key 46
HQ(config)#ntp authentication-key 46 md5 NTBHebat2021
HQ(config)#ntp trusted-key 46
  1. Configure privilege mode password with secret “LksNTB2021”.
HQ(config)#enable secret LksNTB2021
  1. Configure the DNS server of the HQ router with IP address 192.0.2.1.
HQ(config)#ip name-server 192.0.2.1
  1. Enable SSH version 2 on the HQ router with the domain name “mandalika.id” and create RSA keys with a modulus of 1024.
HQ(config)#ip domain-name mandalika.id
HQ(config)#ip ssh version 2
HQ(config)#crypto key generate rsa general-keys modulus 1024
% You already have RSA keys defined named HQ.mandalika.id
% They will be replaced.

% The key modulus size is 1024 bits
% Generating 1024 bit RSA keys, keys will be non-exportable...[OK]
HQ(config)#

  1. Create a local authentication account on the HQ router with username “admin” and password “LksNTB2021”.
    This account is a backup when authentication to the RADIUS server has problems.
HQ(config)#username admin privilege 15 secret LksNTB2021
HQ(config)#line vty 0 4

HQ(config-line)#transport input ssh
HQ(config)#line console 0
HQ(config-line)#login local
HQ(config)#line aux 0
HQ(config-line)#login local
  1. Enable the AAA feature on the HQ router and set the default authentication to use the “local” database.
HQ(config)#aaa new-model
HQ(config)#aaa authentication login default local
  1. Create a user login authentication list with the name “SERVER-AAA” that uses the RADIUS authentication and then use a “local”.
HQ(config)#aaa authentication login SERVER-AAA group RADIUS local
  1. Connect to the RADIUS Server with a second IP address of the subnet 10.0.14.0/30 and use the secret “NTBJawara2021”.
HQ(config)#RADIUS server SERVER-AAA 
HQ(config-RADIUS-server)#address ipv4 10.0.14.2 auth-port 1645
HQ(config-RADIUS-server)#key NTBJawara2021
  1. Configure the HQ router to only accept remote access via SSH with a maximum number of sessions for 5 (five) users at a time and use the “SERVER-AAA” authentication list.
HQ(config)#line vty 0 4
HQ(config-line)#login authentication SERVER-AAA
HQ(config-line)#session-limit 5
  1. Configure console access to the router to use the authentication list “SERVER-AAA”.
HQ(config)#line con 0
HQ(config-line)#login authentication SERVER-AAA
